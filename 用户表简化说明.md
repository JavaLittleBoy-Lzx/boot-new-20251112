# 用户表简化说明

## 📋 简化内容

已将系统用户表从原来的**23个字段**简化为**12个必要字段**。

---

## 🗂️ 简化后的字段列表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 用户ID（主键，自增） |
| username | VARCHAR(50) | 用户名（唯一） |
| password | VARCHAR(255) | 密码（BCrypt加密） |
| role | VARCHAR(20) | 角色（admin/user/guest） |
| status | TINYINT | 状态（0-禁用，1-启用） |
| last_login_time | DATETIME | 最后登录时间 |
| last_login_ip | VARCHAR(50) | 最后登录IP |
| login_count | INT | 登录次数 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |
| is_deleted | TINYINT | 是否删除（0-否，1-是） |
| remark | VARCHAR(500) | 备注 |

## 🔄 功能变化

### 权限管理

**修改前**：
- 每个用户有独立的`permissions`字段存储权限JSON

**修改后**：
- 根据`role`角色自动分配权限：
  - `admin` → `["*"]` - 所有权限
  - `user` → `["read", "write"]` - 读写权限
  - `guest` → `["read"]` - 只读权限

### 用户显示名称

**修改前**：
- 使用`real_name`或`nickname`作为显示名称

**修改后**：
- 直接使用`username`作为显示名称

### 用户唯一性检查

**修改前**：
- 检查用户名、邮箱、手机号唯一性

**修改后**：
- 只检查用户名唯一性

## 🔄 如需扩展

如果将来需要添加更多用户信息，建议：

### 方案1：创建用户详情表
```sql
CREATE TABLE sys_user_profile (
  user_id BIGINT PRIMARY KEY,
  real_name VARCHAR(50),
  email VARCHAR(100),
  phone VARCHAR(20),
  avatar VARCHAR(255),
  department VARCHAR(100),
  ...
  FOREIGN KEY (user_id) REFERENCES sys_user(id)
);
```

### 方案2：使用JSON字段
```sql
ALTER TABLE sys_user 
ADD COLUMN extra_info JSON COMMENT '扩展信息';
```

这样可以保持核心表的简洁性，同时支持扩展功能。
