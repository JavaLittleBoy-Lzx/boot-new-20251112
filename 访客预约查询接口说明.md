# 访客预约查询接口说明

## 接口概述

根据车牌号查询当前有效的访客预约信息，只返回当前时间在 `gateway_transit_begin_time` 和 `gateway_transit_end_time` 之间的预约记录，且只包含非空字段。

## 接口信息

### 基本信息
- **接口路径**: `/parking/nefuData/getVisitorReservationByCarNumber`
- **请求方法**: `GET`
- **Controller**: `VehicleReservationController`

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| carNumber | String | 是 | 车牌号 |

### 请求示例

```http
GET /parking/nefuData/getVisitorReservationByCarNumber?carNumber=京A12345
```

使用 curl:
```bash
curl -X GET "http://www.xuerparking.cn:8080/parking/nefuData/getVisitorReservationByCarNumber?carNumber=京A12345"
```

### 响应字段说明

#### 外层字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| totalCount | Integer | 该车牌号的预约记录总数 |
| validCount | Integer | 当前有效的预约记录数 |
| reservations | Array | 有效的预约记录列表 |

#### reservations 数组中的字段（只包含非空字段）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| reservationId | String | 预约记录ID（用于追踪） |
| visitorName | String | 访客姓名 |
| visitorPhone | String | 访客手机号 |
| passName | String | 被访人姓名 |
| passDep | String | 被访部门名称 |
| vipTypeName | String | 预约VIP类型 |
| gatewayTransitBeginTime | String | 网关通行开始时间 |
| gatewayTransitEndTime | String | 网关通行结束时间 |
| carVisitStatus | String | 车辆来访状态（车未来访/已进场/已离场） |
| carVisitTimes | String | 车辆进出场时间记录（JSON格式） |

**注意**: 
- 只有字段值非空时才会返回该字段
- `carVisitTimes` 是 JSON 字符串格式，包含车辆进出场时间记录

## 业务逻辑说明

### 查询流程

1. **参数校验**: 检查车牌号是否为空
2. **查询数据库**: 从 `visitor_reservation_sync` 表中查询该车牌号的所有未删除记录
3. **时间筛选**: 判断当前时间是否在 `gateway_transit_begin_time` 和 `gateway_transit_end_time` 之间
4. **数据过滤**: 只返回非空字段
5. **返回结果**: 返回符合条件的预约信息列表

### 时间判断逻辑

```java
当前时间 >= gateway_transit_begin_time 
AND 
当前时间 <= gateway_transit_end_time
```

只有同时满足以上条件的预约记录才会被返回。

### 字段过滤规则

对于每个字段，进行以下判断：
- 如果字段值为 `null`，不返回该字段
- 如果字段值为空字符串 `""`，不返回该字段
- 只有字段值非空时才包含在返回结果中

## 使用场景

### 1. 进场车辆验证
当车辆进场时，系统可以调用此接口验证该车辆是否有有效的预约记录。

```javascript
// 前端示例代码
fetch(`/parking/nefuData/getVisitorReservationByCarNumber?carNumber=${carNumber}`)
  .then(res => res.json())
  .then(data => {
    if (data.code === 200 && data.data.validCount > 0) {
      console.log('找到有效预约:', data.data.reservations);
      // 显示访客信息
    } else {
      console.log('无有效预约');
    }
  });
```

### 2. 访客信息展示
在大屏或管理界面显示当前有效的访客预约信息。

### 3. 车辆进出场状态查询
查询车辆的进出场状态和历史记录。

## 注意事项

1. **时间格式**: 所有时间字段均为 `yyyy-MM-dd HH:mm:ss` 格式
2. **数据一致性**: 接口从 `visitor_reservation_sync` 表读取数据，该表由外部接口定期同步
3. **性能考虑**: 接口会查询该车牌号的所有预约记录，对于大量预约记录的情况，建议在数据库层面添加索引
4. **空值处理**: 返回的 JSON 中不包含空值字段，前端需要判断字段是否存在
5. **时区问题**: 当前使用服务器本地时间，需要确保服务器时区设置正确

## 数据库表结构

相关表: `visitor_reservation_sync`

关键字段:
- `car_number`: 车牌号（查询条件）
- `gateway_transit_begin_time`: 网关通行开始时间（时间判断）
- `gateway_transit_end_time`: 网关通行结束时间（时间判断）
- `deleted`: 逻辑删除标记（0-未删除，1-已删除）

建议索引:
```sql
-- 车牌号索引（用于快速查询）
CREATE INDEX idx_car_number ON visitor_reservation_sync(car_number, deleted);

-- 时间范围索引（用于时间筛选）
CREATE INDEX idx_gateway_transit_time ON visitor_reservation_sync(
    gateway_transit_begin_time, 
    gateway_transit_end_time, 
    deleted
);
```
