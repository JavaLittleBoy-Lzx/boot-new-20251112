# 密码验证问题修复说明

## 问题描述

**日期**: 2024-12-06  
**状态**: ✅ 已解决

### 症状
- 用户使用 `admin / 123456` 登录时提示"用户名或密码错误"
- 后端日志显示密码验证失败：`✅ 密码匹配结果: false`

### 根本原因
数据库 `sys_user` 表中存储的 BCrypt 密码哈希值不对应密码 `123456`

```
数据库密码: $2a$10$XcY5rSqQRKdcBqV/VLnvMeMXDWP5qzRKmQgN7OMV2ZW0QJmGbqPXe
测试结果: 所有常见密码都不匹配
```

可能原因：
1. SQL 文件中的哈希值复制错误
2. 使用了不同版本的 BCrypt 库生成
3. 原始密码不是 123456

## 技术细节

### BCrypt 特性
- **算法**: BCrypt with cost factor 10
- **格式**: `$2a$[cost]$[salt][hash]`
- **特点**: 每次生成的哈希值不同（包含随机盐）
- **验证**: 使用 `BCryptPasswordEncoder.matches()` 方法

### 密码验证流程
```
1. 接收登录请求 (username + password)
2. 查询数据库获取用户信息
3. 使用 BCryptPasswordEncoder 验证密码
4. 验证成功 → 生成 JWT Token
5. 验证失败 → 抛出异常
```

## 预防措施

### 1. 密码生成
使用统一的密码生成工具：
```java
BCryptPasswordEncoder encoder = new BCryptPasswordEncoder();
String hash = encoder.encode("原始密码");
```

### 2. 密码验证
始终使用相同的 BCryptPasswordEncoder 实例：
```java
boolean matches = passwordEncoder.matches(rawPassword, encodedPassword);
```

### 3. 数据迁移
- 在迁移 SQL 脚本时，确保密码哈希值完整复制
- 避免手动编辑哈希值
- 迁移后立即测试登录功能

## 总结
