# 🔒 登录密码问题解决方案

## ❌ 问题现象

登录时提示：`用户名或密码错误`

```
密码错误：admin
签发失败：用户名或密码错误
```

---

## 🔍 问题原因

后端虽然已启动，但**数据库中没有用户数据**！

需要先执行数据库初始化脚本创建用户表和初始账号。

---

## ✅ 解决方案

### 方法1：运行初始化脚本（推荐）

1. **双击运行**：`init-database.bat`

2. **脚本会自动执行**：
   - 连接到数据库 `parking_management`
   - 执行 `sql/sys_user.sql`
   - 创建用户表
   - 插入测试账号

3. **成功后显示**：

1. **连接数据库**
   - 主机：`localhost`
   - 端口：`3306`
   - 用户：`root`
   - 密码：`123456`

2. **执行SQL文件**

## 🐛 常见问题

### Q1: 脚本执行失败，提示"未找到MySQL命令"
**原因**：MySQL没有添加到系统环境变量

**解决**：
1. 使用方法2（Navicat等工具）手动执行SQL
2. 或者添加MySQL到PATH环境变量

### Q2: 提示"数据库不存在"
**原因**：数据库 `parking_management` 未创建

**解决**：
```sql
CREATE DATABASE parking_management DEFAULT CHARSET=utf8mb4;
```

### Q3: 仍然提示密码错误
**原因**：可能的情况
1. 数据库脚本未成功执行
2. 后端连接了错误的数据库
3. 用户表中没有数据

**解决**：
```sql
-- 检查用户是否存在
SELECT * FROM sys_user WHERE username = 'admin';

-- 如果没有数据，重新执行插入
INSERT INTO sys_user (username, password, role, status, remark)
VALUES ('admin', '$2a$10$XcY5rSqQRKdcBqV/VLnvMeMXDWP5qzRKmQgN7OMV2ZW0QJmGbqPXe', 'admin', 1, '管理员');
```

### Q4: 后端日志报错
查看后端控制台输出，检查：
- 数据库连接是否成功
- SQL语法是否正确
- 表结构是否正确
