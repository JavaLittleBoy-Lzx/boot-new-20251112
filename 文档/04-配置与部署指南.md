## 三、数据库初始化

### 3.1 创建数据库

```sql
CREATE DATABASE nefu_system 
  DEFAULT CHARACTER SET utf8mb4 
  COLLATE utf8mb4_unicode_ci;
```

### 3.2 执行SQL脚本

按顺序执行以下脚本：

1. **基础表结构**（需要从项目中导出）
2. **添加访问状态字段**：
   ```bash
   mysql -u用户名 -p密码 nefu_system < sql/add_visit_status_fields.sql
   ```

### 3.3 核心表说明

#### visitor_reservation_sync（访客预约同步表）
```sql
CREATE TABLE `visitor_reservation_sync` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `reservation_no` VARCHAR(50) COMMENT '预约编号',
  `visitor_name` VARCHAR(50) COMMENT '访客姓名',
  `visitor_phone` VARCHAR(20) COMMENT '访客手机',
  `visitor_id_card` VARCHAR(20) COMMENT '访客身份证',
  `car_number` VARCHAR(20) COMMENT '车牌号',
  `apply_state_name` VARCHAR(20) COMMENT '预约状态',
  `person_visit_status` VARCHAR(20) COMMENT '人员访问状态',
  `person_visit_times` TEXT COMMENT '人员进出时间JSON',
  `car_visit_status` VARCHAR(20) COMMENT '车辆访问状态',
  `car_visit_times` TEXT COMMENT '车辆进出时间JSON',
  `gateway_transit_begin_time` DATETIME COMMENT '预约开始时间',
  `gateway_transit_end_time` DATETIME COMMENT '预约结束时间',
  `external_create_time` DATETIME COMMENT '外部系统创建时间',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX `idx_visitor_name` (`visitor_name`),
  INDEX `idx_car_number` (`car_number`),
  INDEX `idx_time_range` (`gateway_transit_begin_time`, `gateway_transit_end_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### acms_event_record（ACMS事件记录表）

## 五、部署方式

### 5.1 开发环境启动

#### IDEA启动
1. 打开项目
2. 找到 `ParkingManageApplication.java`
3. 右键 → Run

#### 命令行启动
```bash
mvn spring-boot:run
```

### 5.2 生产环境部署

#### 方式一：直接运行JAR
```bash
java -jar parkingmanage-0.0.1-SNAPSHOT.jar
```

#### 方式二：后台运行
```bash
nohup java -jar parkingmanage-0.0.1-SNAPSHOT.jar > app.log 2>&1 &
```

#### 方式三：使用systemd服务（推荐）

创建服务文件：`/etc/systemd/system/nefu-parking.service`
```ini
[Unit]
Description=NEFU Parking Management System
After=network.target

[Service]
Type=simple
User=app
WorkingDirectory=/usr/local/nefu_system
ExecStart=/usr/bin/java -jar /usr/local/nefu_system/parkingmanage.jar
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl start nefu-parking
sudo systemctl enable nefu-parking  # 开机自启
sudo systemctl status nefu-parking  # 查看状态
```

## 六、Nginx反向代理配置

### 6.1 配置示例

```nginx
server {
    listen 80;
    server_name parking.nefu.edu.cn;
    
    # 重定向到HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name parking.nefu.edu.cn;
    
    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;
    
    # 后端API代理
    location /api/ {
        proxy_pass http://www.xuerparking.cn:8675/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
    
    # 静态文件
    location /uploads/ {
        alias /usr/local/nefu_system/uploads/;
        expires 7d;
    }
    
    # 前端静态资源
    location / {
        root /usr/local/nefu_system/web;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
}
```

### 7.1 日志配置

日志文件位置：`/usr/local/nefu_system/logs/nefuSystem.log`

### 7.2 日志级别

在 `application.yml` 中配置：
```yaml

  level:
    root: INFO
```

### 7.3 日志轮转

使用 `logrotate` 配置文件轮转：

创建 `/etc/logrotate.d/nefu-parking`：
```
/usr/local/nefu_system/logs/*.log {
    daily
    rotate 30
    compress
    delaycompress
    missingok
    notifempty
    create 0644 app app
}
```

## 九、常见问题

### 9.1 启动失败

**问题**：数据库连接失败
```
Could not connect to database
```

**解决**：
1. 检查数据库是否启动
2. 验证用户名密码
3. 确认数据库地址和端口
4. 检查防火墙规则

### 9.2 访客自动同步不工作

**问题**：访客数据未自动同步

**解决**：
1. 检查 `visitor.auto.enabled` 是否为 `true`
2. 查看日志中是否有错误
3. 验证外部API地址是否正确
4. 检查网络连接

### 9.3 海康威视查询失败

**问题**：人员查询返回错误

**解决**：
1. 验证 app-key 和 app-secret
2. 检查海康威视API地址
3. 确认网络可达性
4. 查看详细错误日志

### 9.4 文件上传失败

**问题**：上传文件返回错误

**解决**：
1. 检查上传目录是否存在
2. 验证目录权限
3. 确认文件大小限制配置
4. 检查磁盘空间

## 十一、备份与恢复

### 11.1 数据库备份

**自动备份脚本**：
```bash
#!/bin/bash
BACKUP_DIR="/backup/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
mysqldump -uips -pIps@_123 nefu_system > $BACKUP_DIR/nefu_system_$DATE.sql
# 保留最近30天的备份
find $BACKUP_DIR -name "*.sql" -mtime +30 -delete
```

**设置定时任务**：
```bash
crontab -e
# 每天凌晨2点备份
0 2 * * * /path/to/backup.sh
```

### 11.2 数据库恢复

```bash
mysql -uips -pIps@_123 nefu_system < backup.sql
```

### 11.3 文件备份

```bash
# 备份上传文件
tar -czf uploads_backup.tar.gz /usr/local/nefu_system/uploads/

# 恢复
tar -xzf uploads_backup.tar.gz -C /usr/local/nefu_system/
```
