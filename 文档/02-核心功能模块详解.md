## 一、ACMS数据接收与处理模块

### 1.1 模块概述

**功能**：接收ACMS系统主动推送的人员进出场事件数据

**支持的识别类型**：
- **197162**：人证比对（刷身份证）- 直接使用推送数据
- **196893**：人脸识别 - 需调用海康威视API
- **198914**：刷校园卡 - 需调用海康威视API

**核心接口**：`POST /parking/acms/vip/eventRcv`

### 1.2 数据流程

```
ACMS推送数据
  ↓
解析JSON（支持新旧两种格式）
  ↓
根据eventType判断类型
  ├─ 197162：提取身份证信息
  ├─ 196893：调用海康威视查询
  └─ 198914：调用海康威视查询
  ↓
根据姓名和时间查询预约记录
  ↓
更新person_visit_status和person_visit_times
  ↓
保存事件记录到数据库
  ↓
返回处理结果
```

### 1.3 关键代码

**位置**：`AcmsVipController.java` 第472-974行

```java
@PostMapping("/eventRcv")
public ResponseEntity<Result> eventRcv(@RequestBody String data) {
    // 1. 解析JSON数据
    JSONObject jsonObject = JSON.parseObject(data);
    
    // 2. 判断新旧格式
    JSONArray eventsArray = null;
    if (jsonObject.containsKey("method") && jsonObject.containsKey("params")) {
        // 新格式：从params.events获取
        eventsArray = jsonObject.getJSONObject("params").getJSONArray("events");
    } else {
        // 旧格式：转换为数组
        eventsArray = new JSONArray();
        eventsArray.add(jsonObject);
    }
    
    // 3. 遍历处理每个事件
    for (JSONObject eventObj : eventsArray) {
        Integer eventType = eventObj.getInteger("eventType");
        
        if (eventType == 197162) {
            // 人证比对：直接使用推送数据
            processIdCardEvent(eventObj);
        } else if (eventType == 196893 || eventType == 198914) {
            // 人脸/校园卡：调用海康威视
            String personId = eventData.getString("ExtEventPersonNo");
            PersonInfo personInfo = hikvisionPersonService.queryPersonInfo(personId);
            processPersonEvent(eventObj, personInfo);
        }
    }
    
    return ResponseEntity.ok(Result.success(processResults));
}
```

## 三、访客预约自动同步模块

### 3.1 模块概述

**功能**：定期从外部系统同步访客预约记录，自动更新状态

**核心服务**：`VisitorVipAutoService.java`（57KB）

**调度策略**：智能动态调度，根据时间段自动调整频率

### 3.2 状态机设计

```
┌─────────────┐
│   待来访     │  person_visit_status = "人未来访"
│             │  car_visit_status = "车未来访"
└──────┬──────┘
       │ 进场
       ↓
┌─────────────┐
│   来访中     │  person_visit_status = "人已进场"
│             │  car_visit_status = "已进场"
└──────┬──────┘
       │ 离场
       ↓
┌─────────────┐
│   已签离     │  person_visit_status = "人已离场"
│             │  car_visit_status = "已离场"
└─────────────┘
```

### 3.3 智能调度配置

```yaml
visitor:
  auto:
    schedule:
      mode: dynamic                    # 动态模式
      min-interval: 2000               # 最小2秒
      max-interval: 30000              # 最大30秒
      
      time-based:
        work-hours: "06:00-18:00"      # 工作时间：2秒
        off-hours: "18:00-23:00"       # 下班时间：10秒
        night-hours: "23:00-06:00"     # 夜间：30秒
```

### 3.4 断路器保护

```yaml
circuit-breaker:
  enabled: true
  failure-threshold: 3         # 连续失败3次开启
  timeout-seconds: 60          # 60秒后尝试恢复
  request-timeout: 5000        # 单次请求超时5秒
```

### 3.5 时间记录格式

**person_visit_times** 和 **car_visit_times** 使用JSON数组：

```json
{
  "person_visit_times": [
    {
      "enterTime": "2025-01-15 10:30:00",
      "leaveTime": "2025-01-15 12:00:00"
    },
    {
      "enterTime": "2025-01-15 14:00:00",
      "leaveTime": null   // null表示还未离场
    }
  ]
}
```

### 3.6 核心流程

```
定时任务触发
  ↓
调用外部系统API获取预约列表
  ↓
遍历每条预约记录
  ├─ 判断apply_state_name状态
  │   ├─ "待来访"：设置初始状态
  │   ├─ "来访中"：查询进出场记录，更新状态
  │   └─ "已签离"：查询进出场记录，更新状态
  ↓
查询车辆进场记录（report_car_in）
  ├─ 筛选条件：enterTime > external_create_time
  └─ 获取最新进场记录
  ↓
查询车辆离场记录（report_car_out）
  ├─ 筛选条件：leaveTime > external_create_time
  └─ 获取最新离场记录
  ↓
比较进出时间（时间近似判断：相差<20秒）
  ├─ 有离场记录且时间匹配：更新为"已离场"
  ├─ 只有进场记录：更新为"已进场"
  └─ 无记录：保持"车未来访"
  ↓
保存更新到数据库
```

### 3.7 关键代码

**位置**：`VisitorVipAutoService.java`

```java
@Scheduled(fixedDelay = 2000)  // 2秒执行一次
public void autoSyncTask() {
    if (!autoEnabled) return;
    
    try {
        // 1. 调用外部API获取预约列表
        List<VisitorReservation> reservations = fetchReservationsFromExternal();
        
        // 2. 遍历处理每条记录
        for (VisitorReservation reservation : reservations) {
            // 3. 根据状态判断处理方式
            if ("待来访".equals(reservation.getApplyStateName())) {
                initializeNewReservation(reservation);
            } else if ("来访中".equals(reservation.getApplyStateName())) {
                updateVisitingStatus(reservation);
            } else if ("已签离".equals(reservation.getApplyStateName())) {
                updateCompletedStatus(reservation);
            }
        }
    } catch (Exception e) {
        handleCircuitBreaker(e);
    }
}

private void updateVisitingStatus(VisitorReservation reservation) {
    // 查询最新离场记录
    ReportCarOut latestOut = reportCarOutMapper.selectLatestByCarNumber(
        reservation.getCarNumber(), 
        reservation.getExternalCreateTime()
    );
    
    // 查询最新进场记录
    ReportCarIn latestIn = reportCarInMapper.selectLatestByCarNumber(
        reservation.getCarNumber(),
        reservation.getExternalCreateTime()
    );
    
    // 比较时间，更新状态
    if (latestOut != null && latestIn != null) {
        long timeDiff = Math.abs(
            latestOut.getEnterTime().getTime() - latestIn.getEnterTime().getTime()
        );
        
        if (timeDiff < 20000) {  // 20秒内
            // 时间匹配，已离场
            updateCarVisitStatus(reservation, "已离场", latestOut);
        } else if (latestIn.getEnterTime().after(latestOut.getEnterTime())) {
            // 进场时间更新，已进场
            updateCarVisitStatus(reservation, "已进场", latestIn);
        }
    }
}
```

## 四、车辆进出场数据处理模块

### 4.1 进场数据处理（reportCarIn）

**流程**：
```
接收进场数据
  ↓
根据车牌号查询预约记录
  ↓
根据进场时间筛选（在gateway_transit_begin_time和end_time范围内）
  ↓
更新car_visit_status = "已进场"
  ↓
在car_visit_times JSON数组中添加记录
```

**重复检测**：
- 如果最后一条记录的leaveTime为null
- 且新进场时间与上次相差<1秒
- 判定为重复，不添加

### 4.2 离场数据处理（reportCarOut）

**流程**：
```
接收离场数据
  ↓
根据车牌号查询预约记录
  ↓
从离场记录获取enterTime
  ↓
在car_visit_times中查找匹配的进场记录（时间相差<20秒）
  ↓
更新该记录的leaveTime
  ↓
更新car_visit_status = "已离场"
```

### 4.3 时间匹配规则

**近似时间判断阈值**：
- **20秒**：用于判断进出场记录是否匹配
- **1秒**：用于判断是否为重复记录
- **30分钟**：用于异常情况判断

### 4.4 关键代码

**位置**：`ReportCarInService.java` / `ReportCarOutService.java`

```java
// 进场处理
public void processCarIn(ReportCarIn carIn) {
    // 1. 查询匹配的预约记录
    List<VisitorReservationSync> reservations = 
        visitorReservationSyncMapper.selectByCarNumberAndTimeRange(
            carIn.getCarLicenseNumber(),
            carIn.getEnterTime()
        );
    
    // 2. 更新状态
    for (VisitorReservationSync reservation : reservations) {
        // 检查是否重复
        if (!isDuplicate(reservation, carIn.getEnterTime())) {
            // 添加进场记录
            addCarVisitTime(reservation, carIn.getEnterTime(), null);
            reservation.setCarVisitStatus("已进场");
            visitorReservationSyncMapper.updateById(reservation);
        }
    }
}

// 离场处理
public void processCarOut(ReportCarOut carOut) {
    // 1. 查询匹配的预约记录
    List<VisitorReservationSync> reservations = 
        visitorReservationSyncMapper.selectByCarNumberAndTimeRange(
            carOut.getCarLicenseNumber(),
            carOut.getLeaveTime()
        );
    
    // 2. 更新离场时间
    for (VisitorReservationSync reservation : reservations) {
        // 查找匹配的进场记录
        JSONArray visitTimes = JSON.parseArray(reservation.getCarVisitTimes());
        for (int i = 0; i < visitTimes.size(); i++) {
            JSONObject time = visitTimes.getJSONObject(i);
            if (time.get("leaveTime") == null) {
                long diff = Math.abs(
                    parseTime(time.getString("enterTime")).getTime() - 
                    carOut.getEnterTime().getTime()
                );
                if (diff < 20000) {  // 20秒内匹配
                    time.put("leaveTime", formatTime(carOut.getLeaveTime()));
                    break;
                }
            }
        }
        reservation.setCarVisitTimes(visitTimes.toJSONString());
        reservation.setCarVisitStatus("已离场");
        visitorReservationSyncMapper.updateById(reservation);
    }
}
```
