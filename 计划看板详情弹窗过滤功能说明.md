# 计划看板详情弹窗过滤功能说明

## 修改目的

修改计划看板点击后显示的详情弹窗，使其只显示**已来访**的人员和车辆，而不是全部预约记录。

## 修改内容

### 后端修改

**文件**: `VisitorPlanDashboardController.java`

**方法**: `getDetailList()`（第128-206行）

**修改逻辑**:

#### 1. 访客类型过滤（visitor）
```java
// 访客：只返回人已进场或来访中的记录
reservations = reservations.stream()
    .filter(r -> {
        String status = r.getPersonVisitStatus();
        return StringUtils.hasText(status) && 
               (status.contains("已进场") || status.contains("来访中"));
    })
    .collect(Collectors.toList());
```

**过滤条件**:
- `personVisitStatus` 包含 "已进场"
- 或 `personVisitStatus` 包含 "来访中"

#### 2. 车辆类型过滤（vehicle）
```java
// 车辆：只返回有车牌号且车辆已进场的记录
reservations = reservations.stream()
    .filter(r -> StringUtils.hasText(r.getCarNumber()))
    .filter(r -> {
        String status = r.getCarVisitStatus();
        return StringUtils.hasText(status) && status.contains("已进场");
    })
    .collect(Collectors.toList());
```

**过滤条件**:
- 必须有车牌号（`carNumber` 不为空）
- `carVisitStatus` 包含 "已进场"

## 功能效果

### 修改前
- 点击"计划访客"：显示所有预约的访客（包括未到达的）
- 点击"预计车辆"：显示所有预约的车辆（包括未到达的）

### 修改后
- 点击"计划访客"：**只显示已进场或来访中的访客**
- 点击"预计车辆"：**只显示已进场的车辆**

## 注意事项

1. **状态字段名称**
   - 访客状态字段：`personVisitStatus`
   - 车辆状态字段：`carVisitStatus`

2. **状态值判断**
   - 使用 `contains()` 方法进行模糊匹配
   - 支持"人已进场"、"车已进场"等变体

3. **空值处理**
   - 使用 `StringUtils.hasText()` 检查字段不为空
   - 避免 NullPointerException

4. **数据库字段对应**
   - 确保数据库表 `visitor_reservation_sync` 有对应字段
   - 字段：`person_visit_status`, `car_visit_status`
